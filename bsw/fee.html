Vue d'ensembleOverview

Le Fee émuler une EEPROM au dessus de la Flash interne du MCU. Il convertit les opérations d'écriture bloc-par-bloc en opérations Flash (effacement de secteur + programmation) tout en assurant le wear leveling pour maximiser la durée de vie de la Flash. Il expose l'interface MemIf standard vers le NvM.Fee emulates an EEPROM on top of the MCU's internal Flash. It converts block-by-block write operations into Flash operations (sector erase + programming) while ensuring wear leveling to maximize Flash lifetime. It exposes the standard MemIf interface to NvM.

C — Fee Operations 
/* Fee operates asynchronously:
 *
 * 1. NvM calls Fee_Write(BlockId, Buf)
 *
 * 2. Fee_MainFunction() processes:
 *    a. Find free virtual page
 *    b. Write block header + data
 *    c. If sector full → garbage
 *       collection (copy valid
 *       blocks to next sector,
 *       erase old sector)
 *
 * 3. Fee calls Fls_Write() for
 *    actual Flash programming
 *
 * 4. Fee_MainFunction() polls
 *    Fls_GetJobResult() until done
 *
 * 5. NvM notified via
 *    Fee_JobEndNotification()
 */

/* Wear leveling: Fee alternates
 * between 2+ Flash sectors to
 * distribute erase cycles evenly.
 * Virtual pages track block
 * validity (valid/invalid/erased).
 */

Bonnes pratiquesBest Practices

Conseil : Dimensionnez les secteurs Fee avec au moins 2× la taille totale des blocs pour assurer un garbage collection correct. Utilisez IMMEDIATE-DATA pour les données critiques qui doivent être écrites immédiatement, même pendant un shutdown.Tip: Size Fee sectors to at least 2× the total block size to ensure correct garbage collection. Use IMMEDIATE-DATA for critical data that must be written immediately, even during shutdown.
