Vue d'ensembleOverview

Le Spi est un Handler/Driver qui gère le bus SPI du MCU. Il organise les transferts en 3 niveaux hiérarchiques : Channel (données), Job (ensemble de channels vers un chip select) et Sequence (ensemble de jobs). Il supporte les modes synchrone (polling) et asynchrone (interrupt/DMA).Spi is a Handler/Driver that manages the MCU's SPI bus. It organizes transfers in 3 hierarchical levels: Channel (data), Job (set of channels to one chip select) and Sequence (set of jobs). It supports synchronous (polling) and asynchronous (interrupt/DMA) modes.

C — Spi_AsyncTransmit 
#include "Spi.h"

/* Write data to external EEPROM */
void Eep_WriteBlock(
  uint16 addr, const uint8* data,
  uint16 len)
{
  uint8 cmd[3];

  /* Prepare WRITE command */
  cmd[0] = 0x02U; /* WRITE opcode */
  cmd[1] = (uint8)(addr >> 8);
  cmd[2] = (uint8)(addr);

  /* Setup channels */
  Spi_SetupEB(
    SpiConf_Ch_Command,
    cmd, NULL_PTR, 3U);
  Spi_SetupEB(
    SpiConf_Ch_Data,
    data, NULL_PTR, len);

  /* Start async sequence
   * (DMA + interrupts) */
  Spi_AsyncTransmit(
    SpiConf_Seq_ExtEeprom);

  /* Spi_MainFunction() or ISR
   * completes → notification */
}

/* Check completion */
Spi_SeqResultType res =
  Spi_GetSequenceResult(
    SpiConf_Seq_ExtEeprom);
/* SPI_SEQ_OK / SPI_SEQ_PENDING
 * / SPI_SEQ_FAILED */

Bonnes pratiquesBest Practices

Conseil : Utilisez les External Buffers (EB) pour les gros transferts (pas de copie) et les Internal Buffers (IB) pour les petits commandes. Configurez les priorités de jobs pour éviter les conflits d'accès au bus SPI entre l'EEPROM et le transceiver CAN.Tip: Use External Buffers (EB) for large transfers (no copy) and Internal Buffers (IB) for small commands. Configure job priorities to avoid SPI bus access conflicts between the EEPROM and CAN transceiver.
