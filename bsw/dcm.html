Vue d'ensembleOverview

Le module Dcm implémente la pile diagnostic UDS/OBD. Il gère les sessions diagnostic (Default, Extended, Programming), les niveaux de sécurité, la table de services DSD et le routage des requêtes vers les sous-modules DSP (Data) et DSL (Session Layer).The Dcm module implements the UDS/OBD diagnostic stack. It manages diagnostic sessions (Default, Extended, Programming), security levels, the DSD service table and request routing to the DSP (Data) and DSL (Session Layer) sub-modules.

C — Dcm Callback 
#include "Dcm.h"
#include "NvM.h"

/* DID 0xF190 — Read VIN */
Std_ReturnType Dcm_ReadDid_F190(
  Dcm_OpStatusType OpStatus,
  uint8* Data,
  Dcm_NegativeResponseCodeType* Nrc)
{
  Std_ReturnType ret;

  if (OpStatus == DCM_INITIAL)
  {
    /* Read VIN from NvM block */
    ret = NvM_ReadBlock(
      NvMConf_NvMBlock_VIN,
      Data
    );

    if (ret == E_OK) {
      return E_OK;
    }

    *Nrc =
      DCM_E_CONDITIONSNOTCORRECT;
    return E_NOT_OK;
  }

  *Nrc =
    DCM_E_GENERALREJECT;
  return E_NOT_OK;
}

Bonnes pratiquesBest Practices

Use DcmDspSecurityRow to protect critical services (write, routines) with seed & key. Test each service with a diagnostic tool (CANoe, INCA).
