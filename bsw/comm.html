Vue d'ensembleOverview

Le ComM coordonne les requêtes de communication provenant des SWCs et du Dcm. Il gère l'état de chaque canal réseau (NO_COM → FULL_COM) et interagit avec le Nm pour l'entrée/sortie du réseau. Il notifie le BswM de chaque changement de mode.ComM coordinates communication requests from SWCs and Dcm. It manages the state of each network channel (NO_COM → FULL_COM) and interacts with Nm for network entry/exit. It notifies BswM of each mode change.

C — Application SWC 
#include "ComM.h"
#include "Rte_AppSwc.h"

#define ComMConf_User_App  0U

/* SWC requests full communication */
void App_RequestComm(void)
{
  ComM_RequestComMode(
    ComMConf_User_App,
    COMM_FULL_COMMUNICATION
  );
}

/* SWC releases communication */
void App_ReleaseComm(void)
{
  ComM_RequestComMode(
    ComMConf_User_App,
    COMM_NO_COMMUNICATION
  );
}

/* Mode change notification via RTE */
void App_ComM_ModeNotification(
  ComM_ModeType Mode)
{
  if (Mode == COMM_FULL_COMMUNICATION)
  {
    /* Network is ready — start
     * sending application data */
    App_StartTransmission();
  }
}

Bonnes pratiquesBest Practices

Conseil : Utilisez les PNC (Partial Network Clusters) pour optimiser la consommation énergétique — seuls les nœuds concernés restent actifs. Vérifiez que tous les utilisateurs ComM libèrent leur communication avant l'entrée en sleep.Tip: Use PNC (Partial Network Clusters) to optimize power consumption — only concerned nodes remain active. Verify that all ComM users release communication before entering sleep.
