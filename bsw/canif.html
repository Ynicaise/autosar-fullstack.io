Vue d'ensembleOverview

Le CanIf est la couche d'abstraction entre les modules de communication (PduR, CanTp, CanNm) et le driver CAN (MCAL). Il gère les HRH (Hardware Receive Handles) et HTH (Hardware Transmit Handles), le filtrage des CAN IDs, la gestion des modes du contrôleur (STARTED, STOPPED, SLEEP) et le dispatching des notifications RX vers les modules appropriés.CanIf is the abstraction layer between communication modules (PduR, CanTp, CanNm) and the CAN driver (MCAL). It manages HRHs (Hardware Receive Handles) and HTHs (Hardware Transmit Handles), CAN ID filtering, controller mode management (STARTED, STOPPED, SLEEP) and RX notification dispatching to appropriate modules.

Dépendances principalesKey Dependencies

  • PduR — routage des PDUs en réceptionRX PDU routing

C — CanIf Integration 
/* RX path:
 *
 * 1. CAN controller receives frame
 *    with CAN ID 0x123
 *
 * 2. Can driver ISR calls:
 *    CanIf_RxIndication(
 *      Hrh, CanId, DLC, Data)
 *
 * 3. CanIf matches CAN ID → RxPdu:
 *    CanIfRxPdu_VehSpeed
 *
 * 4. CanIf dispatches to PduR:
 *    PduR_CanIfRxIndication(
 *      RxPduId, PduInfoPtr)
 */

/* TX path: */
#include "CanIf.h"

Std_ReturnType SendFrame(void)
{
  PduInfoType pdu;
  uint8 data[8] = {0};

  pdu.SduDataPtr = data;
  pdu.SduLength = 8U;

  return CanIf_Transmit(
    CanIfConf_TxPdu_VehSpeed,
    &pdu
  );
  /* CanIf queues in HTH buffer,
   * Can driver transmits on bus */
}

Bonnes pratiquesBest Practices

Conseil : Utilisez les Full-CAN HRH pour les trames critiques (filtrage hardware) et les Basic-CAN HRH avec filtrage software pour les trames moins prioritaires. Vérifiez la cohérence entre les CAN IDs configurés dans CanIf et ceux définis dans la matrice de communication (DBC/ARXML).Tip: Use Full-CAN HRH for critical frames (hardware filtering) and Basic-CAN HRH with software filtering for lower-priority frames. Verify consistency between CAN IDs configured in CanIf and those defined in the communication matrix (DBC/ARXML).
