Vue d'ensembleOverview

Le driver Dio fournit un accès direct aux GPIO du MCU. Il lit ou écrit les niveaux logiques des pins (HIGH/LOW) de façon synchrone (pas de buffering ni d'interruption). Le module Port configure les pins (direction, pull-up/down, mode), tandis que Dio ne fait que lire/écrire les niveaux.The Dio driver provides direct access to MCU GPIOs. It reads or writes pin logic levels (HIGH/LOW) synchronously (no buffering or interrupts). The Port module configures pins (direction, pull-up/down, mode), while Dio only reads/writes levels.

C — Dio Read / Write 
#include "Dio.h"

/* Read single channel */
Dio_LevelType level;
level = Dio_ReadChannel(
  DioConf_Ch_LED_Status);
/* STD_HIGH or STD_LOW */

/* Write single channel */
Dio_WriteChannel(
  DioConf_Ch_Relay_Enable,
  STD_HIGH);

/* Toggle (flip) channel */
Dio_FlipChannel(
  DioConf_Ch_LED_Status);

/* Read channel group (4-bit
 * DIP switch on port B[3:0]) */
Dio_PortLevelType grpVal;
grpVal = Dio_ReadChannelGroup(
  &DioConf_Grp_DipSwitch);
/* Returns 0x0..0xF */

/* Write full port */
Dio_WritePort(
  DioConf_Port_B, 0x3000U);
/* Sets PB12=1, PB13=1,
 * others = 0 */

Bonnes pratiquesBest Practices

Conseil : Ne confondez pas Dio (lecture/écriture de niveaux logiques) et Port (configuration de direction, pull-up/down). Nio est synchrone et sans interruption — pour détecter des changements de niveau, utilisez le module ICU ou un CDD avec GPIO interrupt.Tip: Don't confuse Dio (logic level read/write) and Port (direction, pull-up/down configuration). Dio is synchronous with no interrupts — to detect level changes, use the ICU module or a CDD with GPIO interrupt.
