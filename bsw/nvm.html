Vue d'ensembleOverview

Le NvM fournit une abstraction unifiée pour l'accès aux données non-volatiles. Il supporte les blocs natifs, redondants et de type dataset. Il gère la file d'attente des requêtes, la validation CRC, la restauration des ROM defaults et la persistance automatique au shutdown (WriteAll).NvM provides a unified abstraction for non-volatile data access. It supports native, redundant and dataset block types. It manages the request queue, CRC validation, ROM default restoration and automatic shutdown persistence (WriteAll).

C — Application SWC 
#include "NvM.h"

#define NvMConf_Block_EngCalib  8U

typedef struct {
  uint16 ignitionAngle;
  uint16 fuelTrim;
  uint8  boostMap[60];
} EngCalib_T;

static EngCalib_T ramBlock;

/* Read block at startup */
void ReadCalibration(void)
{
  NvM_ReadBlock(
    NvMConf_Block_EngCalib,
    &ramBlock
  );
  /* Async: check status later with
   * NvM_GetErrorStatus() */
}

/* Write after modification */
void SaveCalibration(void)
{
  ramBlock.fuelTrim = 105U;

  NvM_WriteBlock(
    NvMConf_Block_EngCalib,
    &ramBlock
  );
}

/* Job-end callback */
void NvM_EngCalib_JobEnd_Cbk(
  uint8 ServiceId,
  NvM_RequestResultType Result)
{
  if (Result != NVM_REQ_OK) {
    Dem_ReportErrorStatus(
      DemConf_NVM_E_WRITE_FAIL,
      DEM_EVENT_STATUS_FAILED);
  }
}

Bonnes pratiquesBest Practices

Conseil : Utilisez les blocs redondants pour les données critiques (calibration, sécurité). Activez le CRC-32 pour détecter les corruptions. Lors du shutdown, NvM_WriteAll doit avoir suffisamment de temps pour finaliser toutes les écritures en attente.Tip: Use redundant blocks for critical data (calibration, safety). Enable CRC-32 to detect corruption. During shutdown, NvM_WriteAll must have enough time to finalize all pending writes.
