Vue d'ensembleOverview

L'IoHwAb abstrait les I/O hardware (capteurs, actionneurs) pour les SWC applicatifs. Il convertit les valeurs brutes des drivers MCAL (ADC counts, DIO levels, PWM duty) en grandeurs physiques calibrées (tension, température, position). C'est un module ECU-spécifique — pas de standard AUTOSAR pour son contenu, seule l'interface vers le RTE est standardisée.IoHwAb abstracts hardware I/O (sensors, actuators) for application SWCs. It converts raw MCAL driver values (ADC counts, DIO levels, PWM duty) into calibrated physical quantities (voltage, temperature, position). It's an ECU-specific module — no AUTOSAR standard for its content, only the RTE interface is standardized.

C — IoHwAb_ReadSensors.c 
#include "Rte_IoHwAb.h"
#include "Adc.h"

/* ADC calibration constants */
#define ADC_VREF     3.3f
#define ADC_MAX      4095U
#define TEMP_OFFSET  -40.0f
#define TEMP_SCALE   0.1f

void IoHwAb_ReadSensors(void)
{
  Adc_ValueGroupType adcRaw;
  float32 voltage, tempC;

  /* 1. Read raw ADC value */
  Adc_ReadGroup(
    AdcConf_Group_EngTemp,
    &adcRaw
  );

  /* 2. Convert to voltage */
  voltage = (float32)adcRaw
            * ADC_VREF / ADC_MAX;

  /* 3. Convert to temperature */
  tempC = voltage / TEMP_SCALE
          + TEMP_OFFSET;

  /* 4. Provide calibrated value
   *    to SWCs via RTE */
  (void)Rte_Write_PPort_EngTemp(
    tempC
  );
}

/* SWCs see only calibrated °C,
 * not raw ADC counts or channel
 * numbers — full abstraction. */

Bonnes pratiquesBest Practices

Conseil : Centralisez toutes les conversions ADC→physique dans l'IoHwAb. Les SWC applicatifs ne doivent jamais appeler directement les APIs MCAL (Adc_ReadGroup, Dio_ReadChannel). Cela garantit la portabilité ECU et simplifie les tests unitaires des SWC.Tip: Centralize all ADC→physical conversions in IoHwAb. Application SWCs should never directly call MCAL APIs (Adc_ReadGroup, Dio_ReadChannel). This ensures ECU portability and simplifies SWC unit testing.
