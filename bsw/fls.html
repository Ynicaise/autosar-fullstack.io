Vue d'ensembleOverview

Le driver Fls gère les opérations sur la Flash interne du MCU : Fls_Erase() (effacement de secteur), Fls_Write() (programmation page par page) et Fls_Read(). Toutes les opérations sont asynchrones — la progression se fait dans Fls_MainFunction() pour ne pas bloquer l'application pendant l'effacement (qui peut prendre ~100 ms par secteur).The Fls driver manages operations on the MCU's internal Flash: Fls_Erase() (sector erase), Fls_Write() (page-by-page programming) and Fls_Read(). All operations are asynchronous — progress happens in Fls_MainFunction() to avoid blocking the application during erase (which can take ~100 ms per sector).

C — Fls Erase & Write 
#include "Fls.h"

/* Called by Fee for NV storage */

/* Erase a 16 KB sector */
Std_ReturnType FlsErase(void)
{
  return Fls_Erase(
    0x080E0000U,  /* address */
    0x4000U       /* 16 KB */
  );
  /* Fls_MainFunction() processes
   * erase in background
   * (~100 ms for 16 KB sector) */
}

/* Write 64 bytes to Flash */
Std_ReturnType FlsWrite(void)
{
  return Fls_Write(
    0x080E0000U,  /* address */
    srcBuffer,    /* RAM source */
    64U           /* length */
  );
  /* Fls_MainFunction() programs
   * FLS_MAX_WRITE bytes/call */
}

/* Poll or callback */
MemIf_JobResultType res =
  Fls_GetJobResult();
/* MEMIF_JOB_OK
 * MEMIF_JOB_PENDING
 * MEMIF_JOB_FAILED */

Bonnes pratiquesBest Practices

Conseil : Ajustez FLS-MAX-WRITE-NORMAL-MODE pour limiter le temps passé dans Fls_MainFunction() — un trop gros chunk bloque la tâche trop longtemps. Sur certains MCU, l'exécution de code depuis la Flash est impossible pendant l'effacement (read-while-write constraints) : exécutez Fls_MainFunction() depuis la RAM.Tip: Adjust FLS-MAX-WRITE-NORMAL-MODE to limit time spent in Fls_MainFunction() — too large a chunk blocks the task too long. On some MCUs, code execution from Flash is impossible during erase (read-while-write constraints): run Fls_MainFunction() from RAM.
