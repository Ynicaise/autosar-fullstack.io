Vue d'ensembleOverview

Le driver Port configure les pins I/O du MCU au démarrage via Port_Init(). Chaque pin reçoit une direction (INPUT/OUTPUT), un mode (GPIO, CAN_TX, SPI_MOSI, ADC, etc.), un pull-up/down et un drive strength. La plupart des pins sont configurées une seule fois — seules les pins marquées PIN_DIRECTION_CHANGEABLE peuvent être modifiées runtime via Port_SetPinDirection().The Port driver configures MCU I/O pins at startup via Port_Init(). Each pin gets a direction (INPUT/OUTPUT), a mode (GPIO, CAN_TX, SPI_MOSI, ADC, etc.), a pull-up/down and a drive strength. Most pins are configured once — only pins marked PIN_DIRECTION_CHANGEABLE can be modified at runtime via Port_SetPinDirection().

C — Port Initialization 
#include "Port.h"

/* Called once by EcuM during
 * startup — BEFORE any other
 * MCAL driver initialization */
void EcuM_AL_DriverInitOne(void)
{
  /* Configure ALL MCU pins
   * (direction, mode, pull, etc.)
   */
  Port_Init(&PortConfigSet);

  /* After Port_Init:
   * - Pin 42 → CAN TX (AF9)
   * - Pin 28 → GPIO output, LOW
   * - Pin 15 → ADC analog input
   * - Pin 60 → SPI CLK (AF5)
   * ...
   */
}

/* Runtime direction change
 * (only if CHANGEABLE = true) */
Port_SetPinDirection(
  PortConf_Pin_BiDir_Data,
  PORT_PIN_OUT);

/* Change pin mode at runtime */
Port_SetPinMode(
  PortConf_Pin_Debug,
  PORT_PIN_MODE_UART_TX);
/* Only if MODE_CHANGEABLE */

Bonnes pratiquesBest Practices

Conseil : Port_Init() doit être appelé avant tous les autres drivers MCAL (Dio, Spi, Can, Adc...). Configurez les pins non-utilisées en entrée avec pull-down pour éviter les courants de fuite. Limitez les pins DIRECTION_CHANGEABLE au strict nécessaire pour la sécurité.Tip: Port_Init() must be called before all other MCAL drivers (Dio, Spi, Can, Adc...). Configure unused pins as input with pull-down to avoid leakage currents. Limit DIRECTION_CHANGEABLE pins to the strict minimum for safety.
