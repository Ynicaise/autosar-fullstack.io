Vue d'ensembleOverview

Le FrIf abstrait le driver FlexRay pour les couches supérieures. Il gère les clusters FlexRay, la synchronisation des cycles de communication et les job lists qui définissent quand lire/écrire les buffers de messages dans les slots statiques et dynamiques.FrIf abstracts the FlexRay driver for upper layers. It manages FlexRay clusters, communication cycle synchronization and job lists that define when to read/write message buffers in static and dynamic slots.

C — FrIf Job Execution 
/* FrIf_JobListExec_0()
 * called by OS at the start of
 * each FlexRay communication cycle.
 *
 * Sequence:
 * 1. Read RX buffers from static
 *    slots (e.g. slot 3: SensorData)
 *    → PduR_FrIfRxIndication()
 *
 * 2. Write TX buffers to static
 *    slots (e.g. slot 5: BrakeCmd)
 *    ← PduR_FrIfTriggerTransmit()
 *
 * 3. FlexRay controller transmits
 *    during assigned slots in the
 *    5 ms TDMA cycle.
 */

/* FrIf provides deterministic

Bonnes pratiquesBest Practices

Conseil : Les job lists doivent être synchronisées avec le cycle FlexRay. Les slots statiques offrent un accès déterministe (TDMA pur), utilisez-les pour les signaux safety-critical. Les slots dynamiques conviennent aux données événementielles.Tip: Job lists must be synchronized with the FlexRay cycle. Static slots offer deterministic access (pure TDMA), use them for safety-critical signals. Dynamic slots are suitable for event-driven data.
