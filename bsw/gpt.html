Vue d'ensembleOverview

Le driver Gpt pilote les timers hardware du MCU en mode compteur. Il supporte les modes one-shot (délai unique) et continu (périodique). Un canal peut servir de base de temps pour l'OS (system tick), de trigger pour l'ADC, ou de timeout pour des protocoles de communication.The Gpt driver controls MCU hardware timers in counter mode. It supports one-shot (single delay) and continuous (periodic) modes. A channel can serve as OS time base (system tick), ADC trigger, or communication protocol timeout.

C — Gpt Start & Notification 
#include "Gpt.h"

void Gpt_InitTimers(void)
{
  /* Start 1 ms periodic tick
   * 1 MHz × 1000 ticks = 1 ms */
  Gpt_StartTimer(
    GptConf_Ch_OsTick,
    1000U);

  Gpt_EnableNotification(
    GptConf_Ch_OsTick);
}

/* ISR callback every 1 ms */
void Gpt_OsTick_Notification(void)
{
  /* Increment OS counter */
  Os_IncrementCounter(
    OsCounter_SystemTick);

  /* This drives:
   * - OsAlarm activations
   * - Schedule table processing
   * - Task cycle timing
   */
}

/* One-shot timeout example */
Gpt_StartTimer(
  GptConf_Ch_Timeout,
  50000U); /* 50 ms timeout */
/* Notification fires once */

Bonnes pratiquesBest Practices

Conseil : Réservez un timer hardware dédié pour l'OS tick — ne le partagez pas avec d'autres fonctions. La fréquence du tick OS (typiquement 1 ms) détermine la granularité minimale des alarmes et schedule tables.Tip: Reserve a dedicated hardware timer for the OS tick — don't share it with other functions. OS tick frequency (typically 1 ms) determines the minimum granularity of alarms and schedule tables.
