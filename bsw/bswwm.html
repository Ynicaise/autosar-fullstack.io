Vue d'ensembleOverview

Le BswM est l'arbitre de modes de l'ECU. Il reçoit des indications de mode (mode requests) depuis ComM, EcuM, Dcm, NvM et les SWCs, évalue un ensemble de règles configurées et exécute les listes d'actions correspondantes (activer un groupe de PDUs, demander un mode ComM, déclencher un NvM_WriteAll, etc.).BswM is the ECU mode arbiter. It receives mode indications (mode requests) from ComM, EcuM, Dcm, NvM and SWCs, evaluates a set of configured rules and executes the corresponding action lists (activate a PDU group, request a ComM mode, trigger NvM_WriteAll, etc.).

C — Generated / Integration 
/* BswM_MainFunction() — called
 * cyclically by OS (e.g. 10 ms) */

void BswM_MainFunction(void)
{
  /* 1. Evaluate all rules against
   *    current mode indications */

  /* Rule: ComM_FullCom */
  if (BswM_GetComMIndication(
        ComMConf_Channel_CAN0)
      == COMM_FULL_COMMUNICATION)
  {
    /* Execute TRUE action list:
     * Enable PDU groups */
    Com_IpduGroupControl(
      ComConf_IPduGrp_Tx_Vehicle,
      TRUE   /* enable */
    );
    Com_IpduGroupControl(
      ComConf_IPduGrp_Rx_Vehicle,
      TRUE
    );
  }
  else
  {
    /* Execute FALSE action list:
     * Disable PDU groups */
    Com_IpduGroupControl(
      ComConf_IPduGrp_Tx_Vehicle,
      FALSE  /* disable */
    );
  }
}

/* Mode indication from ComM */
void BswM_ComM_CurrentMode(
  NetworkHandleType Network,
  ComM_ModeType Mode)
{
  BswM_SetComMIndication(
    Network, Mode);
}

Bonnes pratiquesBest Practices

Conseil : Documentez chaque règle BswM et sa liste d'actions dans un tableau de traçabilité. Les règles complexes (AND/OR imbriqués) sont difficiles à déboguer — préférez des règles simples et des listes d'actions séquentielles. Testez les transitions de mode avec un simulateur de bus.Tip: Document each BswM rule and its action list in a traceability table. Complex rules (nested AND/OR) are hard to debug — prefer simple rules and sequential action lists. Test mode transitions with a bus simulator.
