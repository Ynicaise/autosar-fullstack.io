Vue d'ensembleOverview

Le driver Icu utilise les canaux timer du MCU en mode capture pour mesurer des signaux d'entrée. Il supporte 4 modes : Signal measurement (période + duty), Timestamp (buffer de timestamps), Edge detection (notification sur front) et Edge counting (compteur d'impulsions). Utilisé par l'IoHwAb pour les capteurs de vitesse, encodeurs et capteurs de position.The Icu driver uses MCU timer channels in capture mode to measure input signals. It supports 4 modes: Signal measurement (period + duty), Timestamp (timestamp buffer), Edge detection (edge notification) and Edge counting (pulse counter). Used by IoHwAb for speed sensors, encoders and position sensors.

C — Icu Measurement 
#include "Icu.h"

/* Measure wheel speed sensor
 * period (rising → rising) */
void ReadWheelSpeed(void)
{
  Icu_ValueType period_ticks;

  Icu_StartSignalMeasurement(
    IcuConf_Ch_WheelSpeed);

  /* After next 2 rising edges... */
  period_ticks =
    Icu_GetTimeElapsed(
      IcuConf_Ch_WheelSpeed);

  /* Convert ticks → Hz
   * Timer clock = 1 MHz
   * f = 1e6 / period_ticks
   *
   * Wheel: 48 teeth
   * RPM = f * 60 / 48
   */
}

/* Edge counting mode */
Icu_StartEdgeCounting(ch);
/* ... wait ... */
Icu_ValueType count =
  Icu_GetEdgeNumbers(ch);
Icu_ResetEdgeCount(ch);

Bonnes pratiquesBest Practices

Conseil : Choisissez le prescaler du timer en fonction de la plage de fréquence attendue. Un prescaler trop faible cause un overflow pour les signaux lents, trop élevé perd la résolution pour les signaux rapides. Pour les capteurs à reluctance variable, filtrez le signal en hardware (filtre RC) avant l'entrée ICU.Tip: Choose the timer prescaler based on expected frequency range. Too low a prescaler causes overflow for slow signals, too high loses resolution for fast signals. For variable reluctance sensors, filter the signal in hardware (RC filter) before the ICU input.
