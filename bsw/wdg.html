Vue d'ensembleOverview

Le driver Wdg configure et alimente le watchdog timer hardware du MCU. Si le logiciel ne trigger pas le watchdog dans le timeout configuré, le MCU effectue un hardware reset. Le WdgM (Services) supervise les entités logiques via le WdgIf (abstraction) qui appelle Wdg_SetTriggerCondition().The Wdg driver configures and feeds the MCU's hardware watchdog timer. If software doesn't trigger the watchdog within the configured timeout, the MCU performs a hardware reset. WdgM (Services) supervises logical entities via WdgIf (abstraction) which calls Wdg_SetTriggerCondition().

ARXML 
<WDG-SETTINGS-CONFIGURATION>
  <SHORT-NAME>
    WdgSettings_Fast
  </SHORT-NAME>
  <WDG-DEFAULT-MODE>
    FAST  <!-- short timeout -->
  </WDG-DEFAULT-MODE>
  <WDG-SETTINGS>
    <WDG-SETTING>
      <MODE>SLOW</MODE>
      <TIMEOUT>0.1</TIMEOUT>
    </WDG-SETTING>
    <WDG-SETTING>
      <MODE>FAST</MODE>
      <TIMEOUT>0.01</TIMEOUT>
    </WDG-SETTING>
    <WDG-SETTING>
      <MODE>OFF</MODE>
      <TIMEOUT>0</TIMEOUT>
    </WDG-SETTING>
  </WDG-SETTINGS>
</WDG-SETTINGS-CONFIGURATION>
C — Wdg Trigger 
#include "Wdg.h"

/* Init watchdog in FAST mode
 * (10 ms timeout) */
Wdg_Init(&WdgConfig);

/* WdgIf calls periodically */
Wdg_SetTriggerCondition(10U);
/* Sets next timeout window:
 * Wdg must be triggered again
 * within 10 ms, else MCU resets.
 *
 * Internally:
 * - Reloads HW wdg counter
 * - Updates timeout value
 */

/* Mode change (e.g. during
 * EcuM shutdown) */
Wdg_SetMode(WDGIF_SLOW_MODE);
/* Extends timeout to 100 ms
 * for shutdown sequence */

/* If no trigger received:
 * HW watchdog expires →
 * MCU RESET → EcuM restarts
 * → DEM logs WDG_RESET event */

Bonnes pratiquesBest Practices

Conseil : Le timeout FAST doit être plus court que le pire-cas de latence de vos tâches critiques. Utilisez le mode SLOW pendant le démarrage et le shutdown (opérations longues). Ne désactivez jamais le watchdog en production — le mode OFF n'est prévu que pour le debug.Tip: FAST timeout must be shorter than the worst-case latency of your critical tasks. Use SLOW mode during startup and shutdown (long operations). Never disable the watchdog in production — OFF mode is only for debugging.
