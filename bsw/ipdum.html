Vue d'ensembleOverview

L'IpduM offre deux fonctionnalités principales : le multiplexage (une partie statique + parties dynamiques sélectionnées par un sélecteur) et la conteneurisation (empaquetage de plusieurs petits PDUs dans un grand container, notamment pour Ethernet/CAN-FD).IpduM provides two main features: multiplexing (a static part + dynamic parts selected by a selector) and containerization (packing multiple small PDUs into a large container, especially for Ethernet/CAN-FD).

Dépendances principalesKey Dependencies

  • Com — fournit les I-PDUs à multiplexerprovides I-PDUs to multiplex
  • PduR — routage du container PDU résultantresulting container PDU routing

C — IpduM Flow 
/* TX Flow with containerization:
 *
 * 1. Com triggers:
 *    PduR_ComTransmit(EngData_PduId)
 *    PduR_ComTransmit(ChassisData_PduId)
 *
 * 2. PduR routes both to IpduM
 *
 * 3. IpduM packs them into one
 *    container PDU (64 bytes):
 *    [Hdr1][EngData][Hdr2][ChassisData]
 *
 * 4. IpduM_MainFunction() or
 *    first-trigger sends container:
 *    PduR_IpduMTransmit(Container_Id)
 *
 * 5. PduR routes to CanIf
 *    → single CAN-FD frame
 */

/* RX side: IpduM receives the
 * container, parses headers and
 * delivers each contained PDU
 * to PduR individually:
 *
 * IpduM_RxIndication(ContainerId)
 *  → PduR_IpduMRxIndication(EngData)
 *  → PduR_IpduMRxIndication(Chassis)
 *  → Com unpacks signals
 */

Bonnes pratiquesBest Practices

Conseil : La conteneurisation est idéale pour maximiser l'utilisation de la bande passante CAN-FD ou Ethernet. Choisissez FIRST-CONTAINED-TRIGGER pour une latence minimale, ou DEFAULT pour attendre que le container soit plein.Tip: Containerization is ideal for maximizing CAN-FD or Ethernet bandwidth usage. Choose FIRST-CONTAINED-TRIGGER for minimal latency, or DEFAULT to wait until the container is full.
