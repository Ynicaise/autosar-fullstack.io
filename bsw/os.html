Vue d'ensembleOverview

L'OS AUTOSAR est un ordonnanceur préemptif basé sur la spécification OSEK/VDX. Il fournit les tâches, alarmes, schedule tables, compteurs, ISR et ressources nécessaires à l'exécution déterministe des runnables BSW et applicatifs. Le RTE mappe les runnables sur les tâches OS.The AUTOSAR OS is a preemptive scheduler based on the OSEK/VDX specification. It provides tasks, alarms, schedule tables, counters, ISRs and resources required for deterministic execution of BSW and application runnables. The RTE maps runnables onto OS tasks.

ARXML 
<OS-TASK>
  <SHORT-NAME>OsTask_10ms</SHORT-NAME>
  <PRIORITY>8</PRIORITY>
  <TASK-TYPE>EXTENDED</TASK-TYPE>
  <SCHEDULE>FULL</SCHEDULE>
  <ACTIVATION>1</ACTIVATION>
  <AUTOSTART>false</AUTOSTART>
  <STACK-SIZE>2048</STACK-SIZE>
</OS-TASK>

<OS-ALARM>
  <SHORT-NAME>
    Alarm_Task_10ms
  </SHORT-NAME>
  <COUNTER-REF>
    SystemCounter
  </COUNTER-REF>
  <ACTION>
    <ACTIVATE-TASK-REF>
      OsTask_10ms
    </ACTIVATE-TASK-REF>
  </ACTION>
  <AUTOSTART>
    <ALARM-TIME>0</ALARM-TIME>
    <CYCLE-TIME>10</CYCLE-TIME>
  </AUTOSTART>
</OS-ALARM>
C — Os_Task_10ms.c 
#include "Os.h"

TASK(OsTask_10ms)
{
  /* RTE-mapped runnables execute here.
   * The RTE generates calls like: */

  /* BSW main functions */
  Com_MainFunctionTx();
  Com_MainFunctionRx();
  BswM_MainFunction();

  /* Application runnables */
  Rte_Runnable_AppSwc_RE_10ms();

  /* Supervision */
  WdgM_CheckpointReached(
    SE_Task10ms, CP_Alive
  );

  TerminateTask();
}

Bonnes pratiquesBest Practices

Conseil : Dimensionnez les piles de tâches avec un facteur de sécurité de 20-30% au dessus du pire cas mesuré. Utilisez les schedule tables plutôt que les alarmes pour les applications à contrainte jitter. Implémentez toujours un ErrorHook pour capturer les erreurs OS.Tip: Size task stacks with a 20-30% safety margin above measured worst case. Use schedule tables instead of alarms for jitter-constrained applications. Always implement an ErrorHook to capture OS errors.
