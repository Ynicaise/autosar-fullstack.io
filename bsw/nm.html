Vue d'ensembleOverview

Le module Nm (Generic NM Interface) fournit une interface unifiée vers les implémentations bus-spécifiques (CanNm, FrNm, UdpNm). Il gère la machine d'état NM et coordonne l'entrée/sortie du réseau entre les différents bus via la synchronisation NM.The Nm module (Generic NM Interface) provides a unified interface to bus-specific implementations (CanNm, FrNm, UdpNm). It manages the NM state machine and coordinates network entry/exit across different buses via NM synchronization.

Dépendances principalesKey Dependencies

  • CanNm / FrNm / UdpNm — implémentations bus-spécifiquesbus-specific implementations

C — NM Integration 
#include "Nm.h"

#define NmConf_Channel_CAN0  0U

/* Called by ComM to enter network */
void EnterNetwork(void)
{
  Nm_NetworkRequest(
    NmConf_Channel_CAN0
  );
  /* NM state: Bus-Sleep → Repeat-Msg
   * NM PDU sent every 500 ms
   * After 1.5 s → Normal Operation */
}

/* Called by ComM to leave network */
void LeaveNetwork(void)
{
  Nm_NetworkRelease(
    NmConf_Channel_CAN0
  );
  /* NM state: Normal → Ready-Sleep
   * If all nodes release:
   *  Wait-Bus-Sleep (2 s) →
   *  Bus-Sleep (transcvr off) */
}

/* NM state change callback */
void Nm_StateChangeNotification(
  NetworkHandleType nmNetworkHandle,
  Nm_StateType nmPreviousState,
  Nm_StateType nmCurrentState)
{
  /* Notify ComM / BswM */
}

Bonnes pratiquesBest Practices

Conseil : Calibrez les timeouts NM en cohérence avec le réseau complet (tous les nœuds doivent partager les mêmes valeurs). Utilisez la coordination NM si l'ECU a plusieurs bus pour garantir un sleep synchronisé.Tip: Calibrate NM timeouts consistently with the full network (all nodes must share the same values). Use NM coordination if the ECU has multiple buses to ensure synchronized sleep.
