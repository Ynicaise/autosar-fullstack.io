Vue d'ensembleOverview

Le Dem centralise le reporting d'erreurs de tous les modules BSW et SWC. Il gère le cycle de vie des DTCs (Diagnostic Trouble Codes) : qualification par debouncing, stockage en mémoire d'événements (primary, secondary, mirror), capture des freeze frames et extended data records.Dem centralizes error reporting from all BSW modules and SWCs. It manages the DTC lifecycle: qualification via debouncing, event memory storage (primary, secondary, mirror), freeze frame capture and extended data records.

Dépendances principalesKey Dependencies

  • NvM — persistance de la mémoire d'événementsevent memory persistence
  • Dcm — lecture / effacement des DTCs via UDSDTC read / clear via UDS
  • FiM — inhibition de fonctions selon les DTCsfunction inhibition based on DTCs

C — SWC / BSW Module 
#include "Dem.h"

/* Generated event ID */
#define DemConf_SensorOverTemp  12U

void MonitorSensorTemp(float32 temp)
{
  if (temp > 120.0f)
  {
    /* Report FAILED — Dem increments
     * the debounce counter. After 3
     * consecutive FAILED reports,
     * the DTC is qualified. */
    Dem_SetEventStatus(
      DemConf_SensorOverTemp,
      DEM_EVENT_STATUS_FAILED
    );
  }
  else
  {
    /* Report PASSED — counter
     * decrements toward -3. */
    Dem_SetEventStatus(
      DemConf_SensorOverTemp,
      DEM_EVENT_STATUS_PASSED
    );
  }
}

/* When DTC qualifies:
 *  - Freeze frame captured
 *  - Stored in primary memory
 *  - Persisted via NvM
 *  - Readable via Dcm (0x19) */

OperationCycle

Un OperationCycle représente une période d'activité significative pour le Dem : cycle d'allumage (ignition cycle), cycle de conduite (driving cycle), cycle OBD, etc. Chaque événement Dem est rattaché à un ou plusieurs cycles via DemOperationCycleRef. Le passage d'un cycle à l'état STARTED ou STOPPED déclenche la réinitialisation des compteurs de debouncing, l'évaluation du healing et l'incrément des compteurs d'aging.An OperationCycle represents a meaningful activity period for the Dem: ignition cycle, driving cycle, OBD cycle, etc. Each Dem event is linked to one or more cycles via DemOperationCycleRef. Transitioning a cycle to STARTED or STOPPED triggers debounce counter reset, healing evaluation, and aging counter increment.

C — EcuM / BswM callback 
#include "Dem.h"

/* Called during ECU startup */
void EcuM_OnGoRun(void)
{
  /* Notify Dem that the ignition
   * cycle has started — resets
   * debounce counters, notifies
   * FiM, triggers healing check */
  Dem_SetOperationCycleState(
    DemConf_DemOperationCycle_IgnitionCycle,
    DEM_CYCLE_STATE_START
  );
}

/* Called during ECU shutdown */
void EcuM_OnGoOffOne(void)
{
  /* Close cycle — triggers aging
   * counter increment for all
   * confirmed + tested events */
  Dem_SetOperationCycleState(
    DemConf_DemOperationCycle_IgnitionCycle,
    DEM_CYCLE_STATE_END
  );
}

Aging & HealingAging & Healing

Healing et Aging sont deux mécanismes distincts de "guérison" d'un DTC :Healing and Aging are two distinct DTC recovery mechanisms:

  • Healing — le DTC passe de CONFIRMED & FAILED à CONFIRMED & PASSED après N rapports PASSED consécutifs dans le même cycle. Il reste en mémoire mais le bit TestFailed est effacé. Contrôlé par DemHealingCycleRef et DemHealingCycleCounterThreshold.Healing — the DTC transitions from CONFIRMED & FAILED to CONFIRMED & PASSED after N consecutive PASSED reports within the same cycle. It stays in memory but the TestFailed bit is cleared. Controlled by DemHealingCycleRef and DemHealingCycleCounterThreshold.
  • Aging — après DemAgingCycleCounterThreshold cycles d'aging complets sans que l'événement soit détecté comme FAILED, le DTC est supprimé de la mémoire d'événements. Nécessite que le DTC soit préalablement healé. Contrôlé par DemAgingCycleRef.Aging — after DemAgingCycleCounterThreshold complete aging cycles without the event being detected as FAILED, the DTC is removed from event memory. Requires the DTC to be previously healed. Controlled by DemAgingCycleRef.

C — Lecture de l'âgeAge Readback 
#include "Dem.h"

/* Read the current aging counter
 * of a DTC (e.g. for engineering
 * purposes or end-of-line test) */
void ReadDTCAgingCounter(void)
{
  uint8 agingCounter = 0U;

  /* Returns number of fault-free
   * cycles elapsed since healing.
   * Range: 0 .. Threshold (40) */
  (void)Dem_GetDTCStatusAvailabilityMask(
    0x900114UL,  /* DTC number */
    DEM_DTC_FORMAT_UDS,
    &agingCounter
  );
}

/* DTC lifecycle summary:
 *
 * FAILED (debounce) ──► CONFIRMED
 *        │
 *   PASSED x1 cycle
 *        │
 *        ▼
 *     HEALED (still in memory)
 *        │
 *   PASSED x40 cycles
 *        │
 *        ▼
 *     AGED (removed from memory) */

Mémoire d'événementsEvent Memory

Le Dem gère jusqu'à trois types de mémoires d'événements indépendantes, chacune configurée avec sa propre capacité et ses propres règles de déplacement (displacement) :The Dem manages up to three independent event memory types, each configured with its own capacity and displacement rules:

TypeTypeRôleRoleConteneur ARXMLARXML Container
PrimaryMémoire principale — DTCs confirmés, freeze frames, extended dataMain memory — confirmed DTCs, freeze frames, extended dataDemPrimaryMemory
SecondaryMémoire secondaire optionnelle — DTCs de priorité inférieure ou OEM-specificOptional secondary memory — lower-priority DTCs or OEM-specificDemSecondaryMemory
MirrorCopie RAM de la mémoire primaire — accès rapide sans lecture NvMRAM copy of primary memory — fast access without NvM readDemMirrorMemory

Lorsque la mémoire est pleine, le Dem applique une politique de displacement : le DTC de plus faible priorité (DemDTCPriority) est écrasé par le nouveau DTC. En cas d'égalité, le plus ancien est déplacé.When memory is full, the Dem applies a displacement policy: the DTC with the lowest priority (DemDTCPriority) is overwritten by the new DTC. In case of tie, the oldest is displaced.

Bonnes pratiquesBest Practices

Conseil : Calibrez les seuils de debouncing en fonction de la périodicité du monitoring. Un seuil trop bas provoque des DTCs intermittents ; trop haut retarde la détection. Utilisez DEM_EVENT_STATUS_PREFAILED/PREPASSED pour un contrôle plus fin.Tip: Calibrate debounce thresholds based on monitoring periodicity. A threshold too low causes intermittent DTCs; too high delays detection. Use DEM_EVENT_STATUS_PREFAILED/PREPASSED for finer control.
