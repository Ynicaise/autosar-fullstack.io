Vue d'ensembleOverview

L'Ea gère l'accès aux blocs de données stockés dans une EEPROM externe (SPI/I²C). Contrairement au Fee (Flash), l'Ea n'a pas besoin de wear leveling car l'EEPROM supporte nativement l'écriture par octet. Il expose l'interface MemIf standard vers le NvM, en parallèle du Fee.Ea manages data block access stored in an external EEPROM (SPI/I²C). Unlike Fee (Flash), Ea doesn't need wear leveling since EEPROM natively supports byte-level writes. It exposes the standard MemIf interface to NvM, alongside Fee.

C — Ea Read / Write 
/* NvM calls Ea via MemIf:
 *
 * Ea_Read(BlockNumber, Offset,
 *         DataBufPtr, Length)
 *
 * Ea_MainFunction() processes:
 *  1. Calculates EEPROM address
 *     from logical block number
 *  2. Calls Eep_Read() (driver)
 *  3. Polls Eep_GetJobResult()
 *  4. Notifies via callback
 *
 * Ea_Write(BlockNumber, DataBufPtr)
 *  → Eep_Write() (byte-level)
 *  → No erase needed (EEPROM)
 */

/* Key difference vs Fee:
 * - No sector erase overhead
 * - No garbage collection
 * - Simpler but slower than Flash
 * - Limited size (typ. 4-64 KB)
 * - Higher endurance (1M+ cycles)
 */

Bonnes pratiquesBest Practices

Conseil : Utilisez l'Ea (EEPROM) pour les petites données à haute endurance (compteurs de cycles, paramètres de sécurité) et le Fee (Flash) pour les grandes données (calibration, DTCs). Vérifiez les temps d'écriture de l'EEPROM externe dans le datasheet.Tip: Use Ea (EEPROM) for small high-endurance data (cycle counters, security parameters) and Fee (Flash) for large data (calibration, DTCs). Check external EEPROM write times in the datasheet.
