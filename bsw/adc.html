Vue d'ensembleOverview

Le driver Adc pilote les unités ADC du MCU. Il organise les canaux en groupes convertis ensemble. Un groupe peut être déclenché par software (one-shot), en continu, ou par hardware trigger (timer, PWM edge). Les résultats sont lus dans un buffer de streaming et transmis à l'IoHwAb pour conversion en valeurs physiques.The Adc driver controls the MCU's ADC units. It organizes channels into groups converted together. A group can be triggered by software (one-shot), continuous, or hardware trigger (timer, PWM edge). Results are read from a streaming buffer and forwarded to IoHwAb for physical value conversion.

C — Adc Conversion 
#include "Adc.h"

Adc_ValueGroupType results[3];

/* Setup result buffer (once) */
Adc_SetupResultBuffer(
  AdcConf_Group_Sensors,
  results);

/* Start HW-triggered group */
Adc_EnableHardwareTrigger(
  AdcConf_Group_Sensors);
/* GPT timer triggers conversion
 * every 1 ms automatically */

/* Notification callback */
void Adc_GrpSensors_Notif(void)
{
  Adc_ReadGroup(
    AdcConf_Group_Sensors,
    results);

  /* results[0] = EngTemp (raw)
   * results[1] = BattVolt (raw)
   * results[2] = ThrottlePos
   *
   * IoHwAb converts:
   * raw → voltage → physical
   * e.g. 2048 → 1.65V → 85°C
   */
}

Bonnes pratiquesBest Practices

Conseil : Utilisez le hardware trigger (GPT) pour les conversions périodiques plutôt que le software trigger dans une tâche OS — cela garantit un timing précis et constant. Groupez les canaux par fréquence d'échantillonnage (rapide vs lent) pour optimiser le temps de conversion.Tip: Use hardware trigger (GPT) for periodic conversions rather than software trigger in an OS task — this ensures precise and consistent timing. Group channels by sampling frequency (fast vs slow) to optimize conversion time.
