Vue d'ensembleOverview

Le driver Fr pilote le contrôleur FlexRay du MCU. Il gère la synchronisation de l'horloge FlexRay (macrotick / microtick), le protocole de démarrage (coldstart node / non-coldstart), l'accès aux message buffers (lecture RX, écriture TX) et le monitoring du Protocol Operation Control (POC) state.The Fr driver controls the MCU's FlexRay controller. It manages FlexRay clock synchronization (macrotick / microtick), startup protocol (coldstart node / non-coldstart), message buffer access (RX read, TX write) and Protocol Operation Control (POC) state monitoring.

C — Fr Startup Sequence 
#include "Fr.h"

/* FlexRay startup sequence */
void Fr_StartCommunication(void)
{
  Fr_POCStatusType pocStatus;

  /* 1. Wakeup on Channel A */
  Fr_SendWUP(FR_CTRL_0);

  /* 2. Wait wakeup complete */
  do {
    Fr_GetPOCStatus(
      FR_CTRL_0, &pocStatus);
  } while (pocStatus.State
    != FR_POCSTATE_READY);

  /* 3. Allow coldstart */
  Fr_AllowColdstart(FR_CTRL_0);

  /* 4. Start communication */
  Fr_StartCommunication(FR_CTRL_0);

  /* POC states:
   * CONFIG → READY → WAKEUP
   * → STARTUP → NORMAL_ACTIVE
   *
   * Coldstart node initiates
   * synchronization, others
   * join the existing cycle. */
}

Bonnes pratiquesBest Practices

Conseil : Au moins 2 coldstart nodes sont requis pour démarrer un cluster FlexRay. Surveillez le POC state NORMAL_PASSIVE qui indique une perte partielle de synchronisation. Les corrections d'horloge (rate/offset) doivent rester dans les limites définies par le cluster.Tip: At least 2 coldstart nodes are required to start a FlexRay cluster. Monitor the NORMAL_PASSIVE POC state indicating partial sync loss. Clock corrections (rate/offset) must stay within cluster-defined limits.
