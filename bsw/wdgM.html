Vue d'ensembleOverview

Le WdgM supervise l'exécution correcte des entités logicielles (Supervised Entities). Il implémente trois mécanismes : alive supervision (vérification de la fréquence d'exécution), deadline monitoring (temps entre deux checkpoints) et logical supervision (enchaînement correct des checkpoints dans un graphe de programme).WdgM supervises the correct execution of software entities (Supervised Entities). It implements three mechanisms: alive supervision (execution frequency check), deadline monitoring (time between two checkpoints) and logical supervision (correct checkpoint sequence in a program flow graph).

Alive Alive Supervision & checkpoint reportingAlive Supervision & Checkpoint Reporting

Configuration d'une entité supervisée avec alive supervision et reporting de checkpoint depuis une tâche cyclique.Supervised entity configuration with alive supervision and checkpoint reporting from a cyclic task.

ARXML 
<WDGM-SUPERVISED-ENTITY>
  <SHORT-NAME>SE_MainTask</SHORT-NAME>
  <SUPERVISED-ENTITY-ID>
    0
  </SUPERVISED-ENTITY-ID>
  <WDGM-CHECKPOINTS>
    <WDGM-CHECKPOINT>
      <SHORT-NAME>CP_Alive</SHORT-NAME>
      <CHECKPOINT-ID>0</CHECKPOINT-ID>
    </WDGM-CHECKPOINT>
  </WDGM-CHECKPOINTS>
</WDGM-SUPERVISED-ENTITY>

<WDGM-ALIVE-SUPERVISION>
  <WDGM-CHECKPOINT-REF>
    CP_Alive
  </WDGM-CHECKPOINT-REF>
  <EXPECTED-ALIVE-INDICATIONS>
    10  <!-- expected per supv cycle -->
  </EXPECTED-ALIVE-INDICATIONS>
  <MIN-MARGIN>2</MIN-MARGIN>
  <MAX-MARGIN>2</MAX-MARGIN>
  <SUPERVISION-REFERENCE-CYCLE>
    100  <!-- ms -->
  </SUPERVISION-REFERENCE-CYCLE>
</WDGM-ALIVE-SUPERVISION>
C — Cyclic Task (10 ms) 
#include "WdgM.h"

#define SE_MainTask_ID   0U
#define CP_Alive_ID      0U

/* Called every 10 ms by OS */
TASK(Task_10ms)
{
  /* 1. Report alive checkpoint */
  WdgM_CheckpointReached(
    SE_MainTask_ID,
    CP_Alive_ID
  );

  /* 2. Application logic */
  App_CyclicProcessing();

  TerminateTask();
}

/* WdgM_MainFunction() runs every
 * 100 ms (supervision cycle).
 * It checks that CP_Alive was
 * reached 10 ±2 times (8..12).
 *
 * If count is out of range:
 *  - SE status → EXPIRED
 *  - Global status → STOPPED
 *  - Wdg not triggered → MCU reset
 */

Bonnes pratiquesBest Practices

Conseil : Utilisez la logical supervision pour les séquences critiques (ex: init → config → run). Les marges alive doivent tenir compte des jitters d'ordonnancement. Testez les scénarios de défaillance (tâche bloquée, boucle infinie) pour valider le temps de réaction du watchdog.Tip: Use logical supervision for critical sequences (e.g. init → config → run). Alive margins must account for scheduling jitter. Test failure scenarios (blocked task, infinite loop) to validate watchdog reaction time.
