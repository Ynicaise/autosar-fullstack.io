Vue d'ensembleOverview

Le LinIf gère les schedule tables LIN (ordonnancement des trames), le mapping des PDUs sur les trames LIN et les transitions de mode (RUN / SLEEP / WAKEUP). Il supporte le rôle Master et coordonne les esclaves via le schedule.LinIf manages LIN schedule tables (frame scheduling), PDU-to-LIN-frame mapping and mode transitions (RUN / SLEEP / WAKEUP). It supports the Master role and coordinates slaves via the schedule.

C — Schedule Switch 
#include "LinIf.h"

/* Switch to diagnostic schedule
 * when Dcm needs LIN diagnostic */
void SwitchToDiagSchedule(void)
{
  LinIf_ScheduleRequest(
    LinIfConf_Channel_0,
    LinIfConf_Schedule_Diag
  );
}

/* Return to normal schedule */
void SwitchToNormalSchedule(void)
{
  LinIf_ScheduleRequest(
    LinIfConf_Channel_0,
    LinIfConf_Schedule_Normal
  );
}

/* LinIf_MainFunction() executes
 * the current schedule:
 * - Sends header for each entry
 * - Lin driver handles sync + PID
 * - Slave responds with data
 * - LinIf notifies PduR
 */

Bonnes pratiquesBest Practices

Conseil : Prévoyez un schedule dédié au diagnostic LIN (MRF/SRF) que BswM active sur demande Dcm. Les délais du schedule doivent être calibrés pour éviter le jitter sur les réponses esclaves.Tip: Plan a dedicated LIN diagnostic schedule (MRF/SRF) that BswM activates on Dcm request. Schedule delays must be calibrated to avoid jitter on slave responses.
