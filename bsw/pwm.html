Vue d'ensembleOverview

Le driver Pwm configure et pilote les canaux timer du MCU en mode PWM. Chaque canal génère un signal avec une période fixe et un rapport cyclique ajustable runtime. L'IoHwAb utilise le Pwm pour contrôler des actionneurs (moteur DC, servo, éclairage LED dimmable).The Pwm driver configures and controls MCU timer channels in PWM mode. Each channel generates a signal with a fixed period and runtime-adjustable duty cycle. IoHwAb uses Pwm to control actuators (DC motor, servo, dimmable LED lighting).

C — Pwm_SetDutyCycle 
#include "Pwm.h"

/* Set fan motor speed via PWM
 * duty cycle (0x0000..0x8000)
 *
 * 0x0000 =   0%  → fan off
 * 0x4000 =  50%  → half speed
 * 0x8000 = 100%  → full speed
 */
void SetFanSpeed(uint8 pct)
{
  uint16 duty;

  /* Scale 0-100% → 0x0000-0x8000 */
  duty = (uint16)(
    ((uint32)pct * 0x8000U) / 100U
  );

  Pwm_SetDutyCycle(
    PwmConf_Ch_FanMotor,
    duty);
}

/* Change period + duty at runtime
 * (for variable-frequency drives)
 */
Pwm_SetPeriodAndDuty(
  PwmConf_Ch_FanMotor,
  2000U,   /* period in ticks */
  0x4000U  /* 50% duty */
);

Bonnes pratiquesBest Practices

Conseil : Configurez le PWM-IDLE-STATE à LOW pour que l'actionneur soit désactivé si le MCU reset. Pour les moteurs, la fréquence PWM doit être au-dessus de la plage audible (> 20 kHz) pour éviter le bruit. Utilisez Pwm_SetOutputToIdle() pour un arrêt propre.Tip: Set PWM-IDLE-STATE to LOW so the actuator is disabled on MCU reset. For motors, PWM frequency should be above audible range (> 20 kHz) to avoid noise. Use Pwm_SetOutputToIdle() for clean shutdown.
