Vue d'ensembleOverview

Le driver Mcu est le premier module MCAL initialisé. Il configure les arbres d'horloge du MCU (oscillateur → PLL → bus clocks), initialise les sections RAM et fournit les services de changement de mode de puissance (RUN → STANDBY/STOP) et de reset contrôlé. Mcu_Init() doit être appelé avant Port_Init() et tous les autres drivers.The Mcu driver is the first MCAL module initialized. It configures MCU clock trees (oscillator → PLL → bus clocks), initializes RAM sections and provides power mode change services (RUN → STANDBY/STOP) and controlled reset. Mcu_Init() must be called before Port_Init() and all other drivers.

C — Mcu Init & Clock 
#include "Mcu.h"

/* Very first MCAL init call */
void EcuM_AL_DriverInitZero(void)
{
  /* 1. Init MCU module */
  Mcu_Init(&McuConfigSet);

  /* 2. Init PLL (8 MHz → 160 MHz)
   */
  Mcu_InitClock(
    McuConf_Clock_160MHz);

  /* 3. Wait PLL lock */
  while (Mcu_GetPllStatus()
    != MCU_PLL_LOCKED) { }

  /* 4. Distribute clock to
   *    all bus peripherals */
  Mcu_DistributePllClock();

  /* After this:
   * HCLK = 160 MHz (AHB)
   * APB1 = 40 MHz
   * APB2 = 80 MHz
   *
   * 5. Init RAM sections */
  Mcu_InitRamSection(0U);
}

/* Shutdown: enter STANDBY */
Mcu_SetMode(
  McuConf_Mode_Standby);
/* MCU enters low-power,
 * only wakeup pins active */

Bonnes pratiquesBest Practices

Conseil : Vérifiez toujours Mcu_GetPllStatus() avant de distribuer l'horloge PLL — un PLL non verrouillé cause des comportements erratiques de tous les périphériques. Utilisez Mcu_GetResetReason() au démarrage pour diagnostiquer les resets inattendus (watchdog, power-on, software).Tip: Always check Mcu_GetPllStatus() before distributing PLL clock — an unlocked PLL causes erratic behavior in all peripherals. Use Mcu_GetResetReason() at startup to diagnose unexpected resets (watchdog, power-on, software).
