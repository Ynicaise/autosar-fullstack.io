Vue d'ensembleOverview

Le driver Eth pilote le contrôleur Ethernet MAC intégré au MCU. Il gère les descripteurs DMA (ring buffers TX/RX), la configuration du PHY via MDIO, le filtrage MAC multicast/broadcast et le timestamping hardware pour IEEE 802.1AS (gPTP). Il supporte 100BASE-T1 (BroadR-Reach) et 1000BASE-T1 pour les réseaux véhiculaires.The Eth driver controls the MCU's integrated Ethernet MAC controller. It manages DMA descriptors (TX/RX ring buffers), PHY configuration via MDIO, multicast/broadcast MAC filtering and hardware timestamping for IEEE 802.1AS (gPTP). It supports 100BASE-T1 (BroadR-Reach) and 1000BASE-T1 for automotive networks.

C — Eth Buffer API 
#include "Eth.h"

/* TX: provide buffer → transmit */
BufReq_ReturnType Eth_TxFlow(void)
{
  uint8* bufPtr;
  Eth_BufIdxType bufIdx;
  uint16 len = 128U;

  /* Get TX buffer from DMA ring */
  Eth_ProvideTxBuffer(
    ETH_CTRL_0, &bufIdx,
    &bufPtr, &len);

  /* Fill Ethernet payload */
  memcpy(bufPtr, payload, len);

  /* Trigger DMA transmission */
  Eth_Transmit(ETH_CTRL_0,
    bufIdx, FRAME_TYPE_IPV4,
    TRUE, len, destMac);

  return BUFREQ_OK;
}

/* RX: ISR → EthIf_RxIndication()

Bonnes pratiquesBest Practices

Conseil : Dimensionnez les buffers DMA en fonction du débit attendu — trop peu de descriptors RX cause des drops sous charge. Pour le TSN/gPTP, activez le timestamping hardware (PTP clock) pour une précision sub-microseconde. Utilisez 100BASE-T1 pour minimiser le câblage véhiculaire.Tip: Size DMA buffers according to expected throughput — too few RX descriptors cause drops under load. For TSN/gPTP, enable hardware timestamping (PTP clock) for sub-microsecond accuracy. Use 100BASE-T1 to minimize automotive wiring.
