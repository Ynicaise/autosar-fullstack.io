Vue d'ensembleOverview

Le PduR route chaque PDU entre sa source et sa destination selon des tables de routage configurées. Il supporte le routage 1:1, 1:N (multicast), le gatewaying inter-bus et le buffering des transmissions.PduR routes each PDU between its source and destination according to configured routing tables. It supports 1:1, 1:N (multicast) routing, inter-bus gatewaying and transmission buffering.

  </PDUR-SRC-PDU>
  <PDUR-DEST-PDUS>

      </SHORT-NAME>
      <DEST-PDU-REF>
        CanIfTxPdu_VehSpeed
      </DEST-PDU-REF>
      <PDUR-DEST-PDU-LO-MODULE>
        CANIF
      </PDUR-DEST-PDU-LO-MODULE>
    </PDUR-DEST-PDU>
  </PDUR-DEST-PDUS>
</PDUR-ROUTING-PATH>
C — PduR Flow 
/* Transmission flow:
 *
 * 1. Com_MainFunctionTx() triggers
 *    PduR_ComTransmit(PduId, PduInfo)
 *
 * 2. PduR looks up routing table:
 *    Src: COM / IPdu_VehSpeed_Tx
 *    Dst: CANIF / CanIfTxPdu_VehSpeed
 *
 * 3. PduR calls:
 *    CanIf_Transmit(TxPduId, PduInfo)
 *
 * 4. CanIf queues the L-PDU into
 *    the CAN driver TX buffer.
 */

/* Reception flow (reverse):
 *
 * 1. CanIf_RxIndication() called
 *    by CAN driver on frame arrival
 *
 * 2. CanIf calls:
 *    PduR_CanIfRxIndication(RxPduId)
 *
 * 3. PduR routes to Com:
 *    Com_RxIndication(ComPduId)
 *
 * 4. Com unpacks signals and
 *    notifies SWCs via RTE.
 */

/* Gateway example: PduR can also
 * route CanIf → LinIf or
 * CanIf → EthIf for bridging. */

Bonnes pratiquesBest Practices

Conseil : Activez le buffering PduR uniquement pour les PDUs gateway ou TP — le buffering systématique consomme de la RAM inutilement. Vérifiez que chaque PDU est routé dans la bonne direction (Tx vs Rx) lors de la revue de configuration.Tip: Enable PduR buffering only for gateway or TP PDUs — systematic buffering wastes RAM. Verify each PDU is routed in the correct direction (Tx vs Rx) during configuration review.
