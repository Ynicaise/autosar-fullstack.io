Vue d'ensembleOverview

L'EcuM gère le cycle de vie de l'ECU. Il initialise le Mcu, le BswM et l'OS au démarrage, coordonne la séquence de shutdown (NvM_WriteAll, ComM déactivation) et configure les sources de wakeup pour les modes sleep.EcuM manages the ECU lifecycle. It initializes Mcu, BswM and OS at startup, coordinates the shutdown sequence (NvM_WriteAll, ComM deactivation) and configures wakeup sources for sleep modes.

Dépendances principalesKey Dependencies

  • Mcu — initialisation des clocks et du resetclock and reset initialization
  • BswM — arbitrage des modesmode arbitration
  • OS — démarrage / arrêt de l'ordonnanceurscheduler start / stop
  • NvM — ReadAll au démarrage, WriteAll au shutdownReadAll at startup, WriteAll at shutdown

  </ECUM-WAKEUP-SOURCE-REFS>
</ECUM-SLEEP-MODE>
C — EcuM Callouts 
#include "EcuM.h"
#include "Mcu.h"
#include "BswM.h"

/* Called by EcuM during STARTUP */
void EcuM_AL_DriverInitOne(
  const EcuM_ConfigType* Config)
{
  /* 1. Init MCU clocks */
  Mcu_Init(Config->McuConfig);
  Mcu_InitClock(McuClockSetting_PLL);

  /* 2. Wait PLL lock */
  while (
    Mcu_GetPllStatus() != MCU_PLL_LOCKED
  ) {}

  Mcu_DistributePllClock();

  /* 3. Init Port, DIO, etc. */
  Port_Init(Config->PortConfig);
  Dio_Init(Config->DioConfig);
}

/* Wakeup validation callback */
void EcuM_CheckValidation(
  EcuM_WakeupSourceType src)
{
  if (src & ECUM_WKSRC_CAN0) {
    /* Validate CAN wakeup frame */
    EcuM_ValidateWakeupEvent(
      ECUM_WKSRC_CAN0
    );
  }
}

Bonnes pratiquesBest Practices

Conseil : La séquence d'initialisation des drivers (callout EcuM_AL_DriverInitOne/Two/Three) doit respecter l'ordre de dépendance des modules. Testez les sources de wakeup en conditions réelles (partial networking, bus idle timeout).Tip: The driver initialization sequence (callout EcuM_AL_DriverInitOne/Two/Three) must respect module dependency order.
