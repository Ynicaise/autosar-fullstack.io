Vue d'ensembleOverview

Le RTE est la couche de communication entre les SWCs et entre les SWCs et le BSW. Il est entièrement généré par l'outil AUTOSAR à partir des descriptions ARXML (SWC, composition système, mapping ECU). Le RTE crée les buffers, les mécanismes de protection (exclusive areas) et les appels Rte_Read/Write/Call/Send/Receive.The RTE is the communication layer between SWCs and between SWCs and the BSW. It is entirely generated by the AUTOSAR tool from ARXML descriptions (SWC, system composition, ECU mapping). The RTE creates buffers, protection mechanisms (exclusive areas) and Rte_Read/Write/Call/Send/Receive calls.

C — Generated Rte_*.c 
/* === Rte_SensorSwc.c (generated) === */

/* Internal RTE buffer */
static uint16 Rte_Buf_Speed = 0U;

Std_ReturnType
Rte_Write_PPort_Speed_value(
  uint16 data)
{
  /* Atomic write to buffer */
  SchM_Enter_Rte_ExclArea();
  Rte_Buf_Speed = data;
  SchM_Exit_Rte_ExclArea();
  return RTE_E_OK;
}

/* === Rte_ControlSwc.c (generated) === */

Std_ReturnType
Rte_Read_RPort_Speed_value(
  uint16* data)
{
  SchM_Enter_Rte_ExclArea();
  *data = Rte_Buf_Speed;
  SchM_Exit_Rte_ExclArea();
  return RTE_E_OK;
}

Bonnes pratiquesBest Practices

Conseil : Ne modifiez jamais le code RTE généré manuellement. Vérifiez les options de génération (mode compatibilité, optimisation des buffers). L'ordre des runnables dans une tâche (POSITION-IN-TASK) impacte la latence de bout en bout.Tip: Never manually modify generated RTE code. Check generation options (compatibility mode, buffer optimization). The runnable order within a task (POSITION-IN-TASK) impacts end-to-end latency.
