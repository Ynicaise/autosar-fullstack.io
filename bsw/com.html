Vue d'ensembleOverview

Le module Com convertit les signaux applicatifs en PDUs et inversement. Il gère le packing / unpacking des signaux dans les I-PDUs, applique les filtres de réception, les valeurs initiales et les notifications au RTE.The Com module converts application signals to PDUs and vice versa. It handles signal packing / unpacking within I-PDUs, applies reception filters, initial values and RTE notifications.

Dépendances principalesKey Dependencies

  • PduR — routage des PDUs en émission / réceptionPDU routing for TX / RX
  • RTE — notifications de réception vers les SWCRX notifications to SWCs
  • OS — appels cycliques Com_MainFunctionTx/Rxcyclic calls to Com_MainFunctionTx/Rx

C — Application 
#include "Com.h"

#define ComConf_Sig_VehicleSpeed  42U

void SendVehicleSpeed(uint16 speed)
{
  uint8 status;

  /* Com packs the signal into the
   * I-PDU at the configured bit
   * position and byte order. */
  status = Com_SendSignal(
    ComConf_Sig_VehicleSpeed,
    &speed
  );

  if (status != E_OK) {
    Dem_ReportErrorStatus(
      DemConf_COM_E_TX,
      DEM_EVENT_STATUS_FAILED
    );
  }
}

/* Com_MainFunctionTx() checks the
 * transmission mode and triggers
 * PduR_ComTransmit() every 10 ms. */
