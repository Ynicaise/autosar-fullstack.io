Explication ARXMLARXML Explained

ARXML (AUTOSAR XML) est le format d'échange standardisé pour toutes les données de configuration et de conception AUTOSAR. Chaque élément d'un système AUTOSAR — des types de données aux descriptions de SWC en passant par les configurations de modules BSW — est décrit dans des fichiers ARXML.ARXML (AUTOSAR XML) is the standardized exchange format for all AUTOSAR configuration and design data. Every element in an AUTOSAR system — from data types to SWC descriptions to BSW module configurations — is described in ARXML files.

Modélisation XDMXDM Modeling

L'approche XDM (XML Data Model) définit la structure et les contraintes des paramètres de configuration des modules AUTOSAR. Elle agit comme le schéma que les outils utilisent pour valider et générer la configuration ARXML.The XDM (XML Data Model) approach defines the structure and constraints for AUTOSAR module configuration parameters. It acts as the schema that tools use to validate and generate ARXML configuration.

Concepts fondamentaux XDMXDM Core Concepts

  • Containers — Éléments de regroupement contenant des paramètres (ex. : ComSignal, CanIfChannel)Containers — Grouping elements that hold parameters (e.g., ComSignal, CanIfChannel)
  • Paramètres — Valeurs typées : entiers, flottants, booléens, énumérations, chaînesParameters — Typed values: integers, floats, booleans, enumerations, strings
  • Références — Liens entre containers (ex. : un signal référençant un PDU)References — Links between containers (e.g., a signal referencing a PDU)
  • Multiplicité — Contraintes de cardinalité (ex. : exactement un EcuMConfig, 0..* ComIPdu)Multiplicity — Cardinality constraints (e.g., exactly one EcuMConfig, 0..* ComIPdu)

Workflow basé sur XDMXDM-Based Workflow

  1. Définir le schéma XDM pour chaque module BSW (définition des paramètres)Define the XDM schema for each BSW module (parameter definition)
  2. Utiliser le schéma pour instancier les configurations en ARXMLUse the schema to instantiate configurations in ARXML
  3. Valider l'ARXML par rapport aux contraintes XDMValidate the ARXML against XDM constraints
  4. Générer le code C à partir de la configuration validéeGenerate C code from the validated configuration
Note : Le XDM est défini par le standard AUTOSAR dans la description des modules BSW. Les outils des fournisseurs (ex. : EB tresos, Vector DaVinci) utilisent le XDM pour fournir des éditeurs de configuration guidés.Note: XDM is defined by the AUTOSAR standard in the BSW Module Description. Vendor tools (e.g., EB tresos, Vector DaVinci) use XDM to provide guided configuration editors.

Mapping RTERTE Mapping

Le RTE (Runtime Environment) est une couche générée qui fournit l'infrastructure de communication entre les SWC et entre les SWC et les modules BSW. Le mapping RTE est le processus de connexion des ports, de mapping des runnables aux tâches et de définition du flux de données.The RTE (Runtime Environment) is a generated layer that provides the communication infrastructure between SWCs and between SWCs and BSW modules. RTE mapping is the process of connecting ports, mapping runnables to tasks, and defining the data flow.

Étapes du mapping RTERTE Mapping Steps

  1. Connexion des ports — Connecter les PPort aux RPort entre SWC dans la compositionPort Connection — Connect PPort to RPort between SWCs in the composition
  2. Mapping des données — Mapper les éléments de données S/R aux chemins de signaux spécifiquesData Mapping — Map S/R data elements to specific signal paths
  3. Mapping des tâches — Assigner chaque runnable à une tâche OS avec un ordonnancement adaptéTask Mapping — Assign each runnable to an OS task with appropriate scheduling
  4. Mapping des zones exclusives — Définir la stratégie de protection de concurrence (verrou d'interruption, verrou de tâche, ressource OS)Exclusive Area Mapping — Define concurrency protection strategy (interrupt lock, task lock, OS resource)
  5. Génération RTE — Générer les fichiers Rte.c / Rte.h à partir du mapping completRTE Generation — Generate the Rte.c / Rte.h files from the complete mapping
