Vue d'ensembleOverview

Un Software Component (SWC) AUTOSAR est le bloc de construction fondamental de la couche applicative. Chaque SWC encapsule une fonctionnalité bien définie et communique exclusivement via des ports définis en ARXML.An AUTOSAR Software Component (SWC) is the fundamental building block of the application layer. Each SWC encapsulates a well-defined piece of functionality and communicates exclusively through ports defined in ARXML.

Exemples de conception — ARXML & Code CDesign Examples — ARXML & C Code

Chaque élément défini en ARXML correspond à du code C généré (ou à implémenter manuellement). Voici les principaux patterns côte à côte.Each element defined in ARXML corresponds to generated C code (or manually implemented code). Here are the main patterns side by side.

Runnable Création d'une Runnable avec TimingEvent (10ms)Creating a Runnable with TimingEvent (10ms)

La description ARXML de la runnable et de son événement déclencheur, et le squelette C correspondant généré par l'outil RTE.The ARXML description of the runnable and its trigger event, and the corresponding C skeleton generated by the RTE tool.

ARXML 
<SWC-INTERNAL-BEHAVIOR>
  <SHORT-NAME>MyCtrl_InternalBehavior</SHORT-NAME>
  <RUNNABLES>
    <RUNNABLE-ENTITY>
      <SHORT-NAME>RE_MyCtrl_10ms</SHORT-NAME>
      <CAN-BE-INVOKED-CONCURRENTLY>
        false
      </CAN-BE-INVOKED-CONCURRENTLY>
      <SYMBOL>RE_MyCtrl_10ms</SYMBOL>
    </RUNNABLE-ENTITY>
  </RUNNABLES>
  <EVENTS>
    <TIMING-EVENT>
      <SHORT-NAME>TE_MyCtrl_10ms</SHORT-NAME>
      <START-ON-EVENT-REF DEST=
        "RUNNABLE-ENTITY">
        RE_MyCtrl_10ms
      </START-ON-EVENT-REF>
      <PERIOD>0.01</PERIOD>
    </TIMING-EVENT>
  </EVENTS>
</SWC-INTERNAL-BEHAVIOR>
C — Rte_MyCtrl.c 
#include "Rte_MyCtrl.h"

/* Runnable: RE_MyCtrl_10ms
 * Trigger : TimingEvent 10ms
 * Task    : OsTask_10ms */
VOID RE_MyCtrl_10ms(VOID)
{
  /* Local variables */
  float32 sensorVal = 0.0f;
  float32 cmdOutput = 0.0f;

  /* 1. Read sensor input via RTE */
  (void)Rte_Read_RPort_SensorVal(
    &sensorVal
  );

  /* 2. Control algorithm */
  cmdOutput = PI_Controller(sensorVal);

  /* 3. Write actuator command */
  (void)Rte_Write_PPort_ActuatorCmd(
    cmdOutput
  );
}

C — Lecture expliciteExplicit read 
/* Generated RTE API prototype
 *   (from Rte_MyCtrl.h) */
Std_ReturnType
  Rte_Read_RPort_SensorVal_SensorVal(
    float32 *data
  );

/* Usage in runnable */
VOID RE_MyCtrl_10ms(VOID)
{
  float32 val;
  Std_ReturnType status;

  status = Rte_Read_
    RPort_SensorVal_SensorVal(&val);

  if (status == RTE_E_OK) {
    /* Process valid data */
    ProcessSensor(val);
  } else {
    /* Use last valid / default */
    Dem_ReportErrorStatus(
      DemConf_SensorTimeout,
      DEM_EVENT_STATUS_FAILED
    );
  }
}

/* Server-side runnable triggered by
 * OperationInvokedEvent */
Std_ReturnType
  RE_DiagSrv_RequestReset(
    uint8 ResetType
  )
{
  Std_ReturnType ret = RTE_E_OK;

  switch (ResetType) {
    case RESET_SOFT:
      EcuM_SelectShutdownTarget(
        ECUM_STATE_RESET,
        ECUM_RESET_MCU
      );
      break;

    case RESET_HARD:
      EcuM_SelectShutdownTarget(
        ECUM_STATE_RESET,
        ECUM_RESET_IO
      );
      break;

    default:
      ret = RTE_E_INVALID;
      break;
  }
  return ret;
}

  <SHORT-NAME>Pim_CtrlState</SHORT-NAME>

    /DataTypes/MyCtrl_StateType

C — Rte_MyCtrl.c 
#include "Rte_MyCtrl.h"

/* Init runnable — called once
 * at SWC startup (InitEvent) */
VOID RE_MyCtrl_Init(VOID)
{
  /* Access Per-Instance Memory */
  MyCtrl_StateType *pim =
    Rte_Pim_Pim_CtrlState();

  /* Set default state */
  pim->mode       = CTRL_MODE_OFF;
  pim->errorCount = 0U;
  pim->lastOutput = 0.0f;
  pim->isReady    = FALSE;

  /* Read calibration parameter */
  float32 kp;
  (void)Rte_Prm_CalPrm_Kp(&kp);
  pim->gainKp = kp;

  /* Signal that init is done */
  (void)Rte_Write_PPort_Status(
    CTRL_STATUS_INITIALIZED
  );
}
Relation ARXML → C : Le fichier Rte_<SWC>.h est généré automatiquement par l'outil RTE à partir des définitions ARXML. Les noms des fonctions RTE (Rte_Read_, Rte_Write_, Rte_Call_, Rte_Pim_) sont déduits des SHORT-NAME ARXML. Le développeur n'implémente que le corps des runnables.ARXML → C Relationship: The Rte_<SWC>.h file is automatically generated by the RTE tool from the ARXML definitions. RTE function names (Rte_Read_, Rte_Write_, Rte_Call_, Rte_Pim_) are derived from the ARXML SHORT-NAMEs. The developer only implements the runnable bodies.

Bonnes pratiquesBest Practices

Bonne pratique : Gardez les SWC cohésifs et à usage unique. Un SWC doit représenter une fonction logique. Évitez les SWC monolithiques gérant plusieurs responsabilités sans rapport.Best Practice: Keep SWCs cohesive and single-purpose. One SWC should represent one logical function. Avoid monolithic SWCs that handle multiple unrelated responsibilities.
Conseil : Utilisez des compositions hiérarchiques (CompositionSwComponentType) pour regrouper des SWC liés et simplifier les connexions de ports au niveau système.Tip: Use hierarchical compositions (CompositionSwComponentType) to group related SWCs and simplify port connections at the system level.
